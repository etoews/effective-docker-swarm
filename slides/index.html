Constraints

tl;dr You don't have access to the Docker Node

  • Cannot use volume mounts
    •
    • Example --volume /node/path:/container/path
      •
    • Exception --volume /var/run/docker.sock:/var/run/docker.sock
      •
    • See Data section
      •
  • Cannot use --privileged, --cap-add, or --cap-drop
    •
  • AppArmor restrictions
    •
  • Try not to docker rm the Swarm management containers
    •
    • But if you do carina rebuild mycluster

$ cat $DOCKER_CERT_PATH/README.md
Using Your Swarm Cluster
========================

This directory contains all of the files the Docker CLI will need to
communicate with your Swarm Cluster. The included files are:

* ca.pem - Certificate Authority, used by clients to validate servers
* cert.pem - Client Certificate, used by clients to identify themselves to servers
* key.pem - Client Private Key, used by clients to encrypt their requests
* ca-key.pem - Certificate Authority Key, private file used to generate more client certificates.
* docker.env - Shell environment config file

How to Whale

  • howtowhale.com oauths with your carina credentials
  • it uses that oauth to create a swarm cluster called howtowhale on ‚Äã*your* account
  • it then runs a jupyter server in a container on that cluster
  • jupyter notebooks run in that container
  • TryDocker is a notebook with a bash kernel, meaning you can run bash commands which are executed on the jupyter server
  • the shell session for TryDocker has the appropriate docker environment variables set so that your docker commands execute against your docker swarm cluster, think DOCKER_HOST and DOCKER_CERT_PATH
